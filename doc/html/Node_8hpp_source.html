121  * @brief a Node is a thread of execution that can suscribe and receive Messages
122  * @details There are two categories of callback functions "Cb" and "Cfg" and
123  * they are called from different threads.
124  * All messages are received through "Cb" callback functions. All "Cb"callback functions
125  * are called in this Node's thread sequentially, so there is no data protection needs
126  * within a Node from this perspective.
127  * The framework uses "Cfg" callback functions mostly at the initialization stage to set up
128  * pub/sub configurations before message dispatching
129  * @tparam CcNode The concrete Node type
130  * @tparam RecvMessageTuple The std tuple list all the received Message types.
131  * The matching handleMessageCb for the above type needs to be provided for each type
132  * so this message is handled - othewise cannot not compile. For example:
133  * void handleMessageCb(MessageA const& m){...}
